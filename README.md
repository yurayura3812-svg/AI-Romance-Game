## 概要

大規模言語モデル（LLM）を活用した恋愛シミュレーションゲームです。
AIキャラクターとの自然な会話、感情に応じた表情変化、好感度システムを実装しています。
授業の4人チームで制作。コード実装・システム統合・バグ修正を担当。

---

## 技術的なポイント

### 1. LLMを用いたキャラクターAI
- Groq API（llama-3.3-70b）を使用したリアルタイム会話生成
- キャラクターの性格・口調・状況をシステムプロンプトで制御
- 好感度レベルに応じてプロンプトを動的に変化させることで、関係性の深まりを表現

### 2. 感情分類による表情変化
- AIの応答テキストをLLMで感情分析（normal / happy / blush / sad / angry）
- 感情ラベルに対応した立ち絵画像をリアルタイムで切り替え
- テキストと表情の一致によって没入感を向上

### 3. 好感度評価システム
- LLMによる会話評価とキーワード評価を組み合わせたハイブリッド方式
- 好感度に応じてキャラクターの呼び方・態度・口調が変化
- エンディングでの好感度推移グラフで、プレイ体験を可視化

### 4. イベントシステム
- 好感度を条件としたイベント解放（花見・夏祭り・クリスマス・告白）
- イベントごとに背景・衣装・プロンプトを切り替え
- 告白成功はLLMによる応答評価で判定

## 使用技術

| 分類 | 技術 |
|------|------|
| フロントエンド | Streamlit |
| LLM API | Groq API（llama-3.3-70b-versatile） |
| 画像処理 | Pillow（背景とキャラクター立ち絵の合成） |
| データ可視化 | pandas / Streamlit line_chart |
| デプロイ | Streamlit Cloud |

---

## 技術選定理由

| 技術 | 選定理由 |
|------|----------|
| Groq API | 無料枠があり個人開発に導入しやすい。また入力データが学習に使用されないためキャラクター設定等のデータを安全に扱える。推論速度が速くゲームのレスポンス体験を損なわない |
| Streamlit | PythonのみでUIを構築できるためフロントエンドの実装コストを抑えられる。Streamlit Cloudへの無料デプロイが容易 |
| Pillow | 背景画像とキャラクター立ち絵を動的に合成するため。軽量で追加依存が少ない |
| Ollama（開発時） | ローカルで無料に動かせるためAPI費用を気にせず開発・デバッグができる |

## システム構成

    プレイヤー入力
        ↓
    会話生成（Groq API）
        ↓
    好感度評価（LLM + キーワード） / 感情分類（LLM）
        ↓
    表情・プロンプト・イベント条件を更新
        ↓
    画面に反映（立ち絵・背景・セリフ）

## ローカル実行

**1. リポジトリをクローン**

    git clone https://github.com/yurayura3812-svg/AI-Romance-Game.git
    cd AI-Romance-Game
    pip install -r requirements.txt

**2. `.streamlit/secrets.toml` を作成**

    GROQ_API_KEY = "your_groq_api_key"

**3. 起動**

    streamlit run app.py
